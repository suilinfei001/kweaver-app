# Browse

Headless browser for QA testing and site verification.

## When to Use

- User asks to "open in browser"
- Testing web applications
- Verifying deployments
- Taking screenshots
- Testing user flows

## Capabilities

- Navigate URLs
- Interact with elements
- Verify page state
- Take annotated screenshots
- Test responsive layouts
- Handle forms and uploads
- Assert element states
