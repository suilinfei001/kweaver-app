# Brainstorming

Technical exploration and design documentation workflow.

## When to Use

- Before `/writing-plans` or `/plan-eng-review`
- Exploring technical approaches
- Creating design documents and specs
- Open-ended technical investigation

## Process

1. Understand user intent and requirements
2. Research and explore options
3. Generate design document or spec
4. Get feedback and iterate
