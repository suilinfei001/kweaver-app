# Investigate

Systematic debugging workflow for investigating bugs and issues.

## When to Use

- User reports a bug or error
- App crashes or throws exceptions
- Unexpected behavior observed
- "It was working yesterday" type issues

## Process

1. **Phase 1: Investigate** - Gather evidence, reproduce the issue
2. **Phase 2: Analyze** - Identify patterns and potential causes
3. **Phase 3: Hypothesize** - Form testable hypotheses
4. **Phase 4: Implement** - Fix the root cause

**Iron Rule**: No fixes without root cause. Phase 1 must complete before Phase 4 begins.
