# Frontend Design

UI design workflow ensuring consistent design quality.

## When to Use

- User asks for UI design
- Implementing new Compose screens
- Designing complex layouts
- Redesigning existing pages

## Process

1. Generate design proposal using skill
2. Review and get user feedback
3. Implement according to approved design
4. Verify implementation matches design

## Requirements

- All UI elements must have `contentDescription` or `testTag` for E2E testing
- Follow existing Compose patterns in codebase
- Use theme colors from Material Design 3
