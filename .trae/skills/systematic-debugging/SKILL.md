# Systematic Debugging

Four-phase debugging methodology for root cause analysis.

## When to Use

- Debugging any bug or issue
- Before proposing any fix
- Troubleshooting unexpected behavior
- "Why is this broken?" questions

## Four Phases

1. **Investigate** - Gather evidence, reproduce the issue
2. **Analyze** - Identify patterns and potential causes
3. **Hypothesize** - Form testable hypotheses
4. **Implement** - Fix the root cause

**Iron Rule**: Phase 1 must complete before Phase 4 begins. No root cause = no fix.
